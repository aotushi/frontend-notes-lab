script async / defer 验证

打开控制台或查看下方日志。为了稳定验证 async 的加载顺序，请用同目录的 server.js 启动本地服务；直接用 file:// 打开时没有人为网络延迟。

可验证结论
  1. 无属性外链脚本会阻塞 HTML 解析，执行时 document.readyState 仍是 loading。
  2. async 外链脚本下载完就执行，不保证按标签顺序。此示例中 slow 标签在前，但 fast 会先打印。
  3. defer 外链脚本等 HTML 解析完成后执行，并且按标签顺序执行。此示例中 A 标签在前，即使响应更慢，也会先于 B 打印。
  4. DOMContentLoaded 会在所有 defer 脚本执行完成后触发。
